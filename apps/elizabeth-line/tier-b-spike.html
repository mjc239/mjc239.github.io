Tier B feasibility spike

Checks whether TfL's ArrivalDepartures feed (a) returns live data for Elizabeth line stations and (b) has scheduledTimeOfDeparture values that line up with Journey Planner — the join Tier B would rely on. Runs entirely in your browser through the app's proxy. Nothing is saved or sent anywhere else.

Running… give it a few seconds.

Read it like this: a station is Tier-B-ready if ArrivalDepartures returns rows and most Journey Planner departures find a scheduled-time match. If outer branches come back empty, Tier B would only work for the core — useful to know before building it.
